🧩 PART 2 — MERGE INTO ONE MASTER FILE

Create this file:

/assets/system-evolution/merge.txt

Inside it:

file 'making-of-panel.mp4'
file 'panel-to-inverter.mp4'
file 'inverter-to-battery.mp4'
file 'battery-to-system.mp4'

Now run in terminal:

cd assets/system-evolution/raw

ffmpeg -f concat -safe 0 -i ../merge.txt -c copy system-evolution-master.mp4

Now you have:

/assets/system-evolution/raw/system-evolution-master.mp4

This is your cinematic master.

🧩 PART 3 — CREATE FRAME OUTPUT FOLDER

Now we move to your Next.js public directory.

Create:

/public/frames/system-evolution/

This is important.

Anything inside /public becomes accessible via URL:

/frames/system-evolution/frame_0001.webp
🧩 PART 4 — EXTRACT FRAMES

Now extract frames from master video.

Inside /assets/system-evolution/raw/ run:

ffmpeg -i system-evolution-master.mp4 -vf "fps=30,scale=1920:-1" frame_%04d.png

This does:

• fps=30 → 30 frames per second
• scale=1920:-1 → keep aspect ratio, width 1920
• frame_%04d → names like frame_0001.png

If video = 20 seconds
20 × 30 = 600 frames

You’ll now see:

frame_0001.png
frame_0002.png
...
frame_0600.png
🧩 PART 5 — CONVERT TO WEBP (Critical for performance)

PNG is too heavy.

Run:

for f in frame_*.png; do
  name=$(basename "$f" .png)
  cwebp -q 80 "$f" -o "${name}.webp"
done

Now you get:

frame_0001.webp
frame_0002.webp
...

Delete PNG files after.

🧩 PART 6 — MOVE FRAMES TO PUBLIC

Redraw canvas only when frame index changes.
🧩 PART 8 — HOW SCROLL MAPPING WORKS (Conceptually)

Think of scroll like this:

Scroll top → progress = 0
Scroll bottom → progress = 1

Frame index =

Math.floor(progress * (totalFrames - 1))

So if:

progress = 0.5
frameIndex = 0.5 × 599 = ~300

That’s middle of sequence.

This is how your hero works too.
